How can I export my graphs?

You can export your graphs as PNG or SVG. You can choose the size of the image so that it fits perfectly in your document.

What do "A4 full width" and "A4 half width" on the exporting size menu mean?

Microsoft Word, Libre Office, Google Docs and other text editors usually use a default image resolution of 96ppi (pixels per inch). An A4 page has 8.27 inches wide, so leaving 2 inches for left and right margins, this gives 6.27 inches or approximately 602 pixels wide. This is the "A4 full width" exporting size, with a height chosen to give a nice aspect ratio for an image in a document. The "A4 half width" is a little bit less than 602/2 pixels, so as to leave a small gap in case you want to include two images side by side.

But how do I use a SVG file?

SVG files are the most standard format for vector graphics. A SVG file, as opposed to PNG files, lets you "zoom in" the image infinitely without "loosing resolution". To see and edit SVG files you can use the open source program InkScape. Here is an example (try zooming in):

But what are they good for? If you use LaTex and are a perfectionist when it comes to writing documents, you might already know. When you open a PDF and see an image that looks perfect, regardless of how much zoom-in, you can tell it is a vector image.

So how do you place a SVG in LaTex, you might ask? The option is not available by default. You have to include the package \usepackage{svg} and then include an image with the command \includesvg{path/figure.svg} and that's it!

My graphs always have a white background, can't I have a transparent background?

We include a white background to avoid some undesired results when you plot areas and lines, which can land outside the plotting area. There is a workaround though, you can save your graph as SVG and remove the background with InkScape.
